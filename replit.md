# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

See the `pnpm-workspace` skill for workspace structure, TypeScript setup, and package details.

## DashFlow (`artifacts/newtab-dashboard`)

A premium Chrome Extension New Tab dashboard built with React + TypeScript + Vite + TailwindCSS 4.

### Key files
- `src/components/dashboard/NewTabDashboard.tsx` — main dashboard component
- `src/components/dashboard/store.ts` — all Zustand stores: Grid, Theme, Todo (priority+dnd), QuickLinks, Bookmarks, Calendar, Notes
- `src/components/dashboard/grid.tsx` — dnd-kit drag-and-drop grid engine
- `src/components/dashboard/widgets.tsx` — Clock, Weather, Todo (enhanced), Quick Links, Bookmarks, Calendar, Notes
- `src/store/storage.ts` — Chrome Storage persistence (local + sync)
- `src/hooks/useWeather.ts` — OpenWeatherMap API hook
- `src/components/weather/WeatherIcon.tsx` — animated SVG weather icons
- `wxt.config.ts` — WXT Chrome Extension configuration
- `entrypoints/newtab/` — Extension New Tab page entry
- `entrypoints/popup/` — Extension toolbar popup

### Widgets
| Widget | Features |
|--------|----------|
| Clock | Live time, seconds bar, day/date, dark/light |
| Weather | OWM API, geolocation, city search, °C/°F, 5-day forecast |
| Quick Links | 6 icons (GitHub, Figma, Notion, Linear, Vercel, Slack), persisted |
| Bookmarks | Chrome bookmarks list with color tags, persisted |
| Todo | Priority dots (low/med/high), drag-to-sort (dnd-kit), deadline field, All/Active/Done tabs |
| Calendar | Monthly grid, today highlight, event dots, upcoming list, add-event inline form |
| Notes | Multi-tab (up to 5), auto-save, markdown-lite (**bold**, `code`, - bullets), word/char count |

### Data persistence
All stores use `chrome.storage.local` (or `localStorage` in web mode) via `STORAGE_KEYS`:
- `newtab_layouts`, `newtab_theme`, `newtab_todos`, `newtab_quicklinks`, `newtab_bookmarks`
- `newtab_calendar` (events array), `newtab_notes` ({notes, activeNoteId})
- `newtab_weather_key`, `newtab_weather_city`, `newtab_weather_unit`

### Scripts
- `pnpm dev` — Vite dev server (Replit preview, hot-reload)
- `pnpm build` — WXT extension build → `.output/chrome-mv3/` (load unpacked in Chrome)
- `pnpm dev:ext` — WXT dev mode (for extension hot-reload in Chrome)
- `pnpm zip` — Package extension as `.zip` for distribution
- `pnpm build:web` — Vite web build (for hosted web app)

### Chrome Extension structure
The extension manifest (MV3) is auto-generated by WXT:
- `chrome_url_overrides.newtab` → dashboard replaces Chrome's new tab
- `action.default_popup` → toolbar icon opens compact popup
- Permissions: `storage`, `bookmarks`, `geolocation`
- CSP allows: Google Fonts, OpenWeatherMap API

### Loading the extension in Chrome
1. Run `pnpm --filter @workspace/newtab-dashboard run build`
2. Open `chrome://extensions`
3. Enable "Developer mode"
4. Click "Load unpacked" → select `artifacts/newtab-dashboard/.output/chrome-mv3/`
